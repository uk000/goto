# MCP Server Features
The `pkg.mcp/server` sub-package implements a generic MCP server with admin APIs to let users configure `Goto` as one or more MCP agents exposed over same or different ports.

The `MCP` admin APIs are exposed with path-prefix `/mcpapi/...` and allow users to add/start/stop MCP servers, add tools/resources to an MCP server, and get MCP server details.

Once an MCP server is configured, a client can call `/mcp/{server-name}` to interact with the MCP server over MCP protocol.

# MCP Admin APIs
These APIs are used to add/remove/configure MCP servers and their components (tools, prompts, resources, resource templates) to be exposed by `Goto`.

# <a name="mcp-server-apis"></a>

## Server Management

## Servers APIs (apply to all MCP servers configured for a port)

|METHOD|URI|Description|
|---|---|---|
| GET | /mcpapi/servers | Get details of the configured MCP servers on the current port. Returns JSON by default, or YAML if `Accept: application/yaml` header is set. |
| GET | /mcpapi/servers/all | Get details of all configured MCP servers across all ports. Returns JSON by default, or YAML if `Accept: application/yaml` header is set. |
| GET | /mcpapi/servers/names | Get a summary map of MCP server names to their tool names, grouped by port. |
| POST | /mcpapi/servers/add | Add one or more MCP servers from the request's JSON array payload. [See MCPServerPayload Schema](#mcpserverpayload) |
| POST | /mcpapi/servers/start | Enable all MCP servers on the current port (servers are enabled by default). |
| POST | /mcpapi/servers/stop | Disable all MCP servers on the current port to prevent them from serving MCP requests. |
| POST | /mcpapi/servers/clear | Clear all component state (tools, prompts, resources, templates, payloads, sessions) from all servers on the current port. |
| POST | /mcpapi/servers/clear/all | Remove all MCP servers across all ports, resetting global state. |
| GET | /mcpapi/servers/{kind} | Get all components of the given `{kind}` from all MCP servers across all ports. `{kind}` must be one of `tools`, `prompts`, `resources`, `templates`. Returns JSON by default, or YAML if `Accept: application/yaml` header is set. |
| GET | /mcpapi/servers/status | Get all configured MCP status rules across all ports. |
| POST | /mcpapi/servers/status/clear | Clear all configured MCP status rules on the current port. |


## Server APIs (apply to a specific MCP server on a port)
|METHOD|URI|Description|
|---|---|---|
| GET | /mcpapi/server/{server} | Get details of a specific MCP server by name. Returns `400` if the server is not found on any port. Returns JSON by default, or YAML if `Accept: application/yaml` header is set. |
| POST | /mcpapi/server/{server}/route?uri={uri} | Configure the given MCP server to be served over the specified URI. All tools of the server are also registered on their corresponding subpaths under the given URI. An SSE URI is auto-generated by inserting `/sse` into the path after `/mcp`. |
| POST | /mcpapi/server/{server}/start | Mark a specific MCP server as enabled (servers are enabled by default). |
| POST | /mcpapi/server/{server}/stop | Mark a specific MCP server as disabled to prevent it from serving MCP requests. |
| POST | /mcpapi/server/{server}/clear | Clear all component state from a specific MCP server. Returns `400` if the server is not found. |


#### A Server's Component Management (Tools, Prompts, Resources, Templates)

|METHOD|URI|Description|
|---|---|---|
| POST | /mcpapi/server/{server}/{kind}/add | Add one or more components of the given `{kind}` to the specified server. `{kind}` must be one of `tools`, `prompts`, `resources`, `templates`. Accepts a JSON array of component objects. See [MCPTool](#mcptool), [MCPPrompt](#mcpprompt), [MCPResource](#mcpresource), [MCPResourceTemplate](#mcpresourcetemplate) schemas. |
| GET | /mcpapi/server/{server}/{kind} | Get all components of the given `{kind}` from a specific MCP server. Returns `400` if the server is not found. Returns JSON by default, or YAML if `Accept: application/yaml` header is set. |
| GET | /mcpapi/server/{server}/{kind}/{name} | Get a specific named component of the given `{kind}` from a specific MCP server. Returns JSON by default, or YAML if `Accept: application/yaml` header is set. |

#### Component Payload Configuration

|METHOD|URI|Description|
|---|---|---|
| POST | /mcpapi/server/{server}/payload/{kind}/{name} | Set a static response payload for a component. `{kind}` must be one of `tools`, `prompts`, `resources`, `templates`. `{name}` is the component name. Sends `Content-Type: application/json` for JSON payloads, or other content types for text. |
| POST | /mcpapi/server/{server}/payload/{kind}/{name}/stream/count={count} | Set a streaming response payload for a component. The payload body is split into `{count}` chunks and streamed back on invocation. |
| POST | /mcpapi/server/{server}/payload/{kind}/{name}/stream/count={count}/delay={delay} | Set a streaming response payload with inter-chunk delay. `{delay}` format: `<min>-<max>` or `<min>-<max>:<delayCount>`, e.g. `100ms-500ms` or `1s-2s:5`. See [Delay format](#delay). |
| POST | /mcpapi/server/{server}/payload/completion?type={type} | Set a completion payload for the given server. The `type` query param specifies the completion reference type. The body is text content split into suggestion lines. |
| POST | /mcpapi/server/{server}/payload/completion/delay={delay}?type={type} | Set a completion payload with inter-suggestion delay. `{delay}` format: `<min>-<max>` or `<min>-<max>:<delayCount>`. See [Delay format](#delay). |

#### Tool Call (server-side)

|METHOD|URI|Description|
|---|---|---|
| POST | /mcpapi/server/{server}/tool/{tool}/call | Directly invoke a tool on the specified MCP server via REST. Accepts a JSON object as the tool arguments in the request body. Returns the tool's result as JSON. Returns `400` if the server or tool is not found. |

# <a name="mcp-json-schemas"></a>

#### MCPServerPayload
<a name="mcpserverpayload"></a>

The `/mcpapi/servers/add` API accepts a JSON array of `MCPServerPayload` objects: `[MCPServerPayload, ...]`

|Field|Data Type|Required|Description|
|---|---|---|---|
| port | int | Yes | The port over which the MCP server will be exposed. `Goto` must be configured to open an RPC protocol listener on this port. |
| name | string | Yes | MCP server name. Used to auto-generate server URI, and referenced by other APIs for further configuration (e.g. adding tools). |
| version | string | Yes | Server version string, e.g. `"1.0"`. |
| description | string | No | Human-readable description of the MCP server. |
| instructions | string | No | Instructions text provided to MCP clients during initialization. |
| keepAlive | duration | No | Keep-alive duration for server sessions, e.g. `"30s"`, `"5m"`. |
| pageSize | int | No | Page size for paginated list responses (tools, prompts, resources). |
| hasPrompts | bool | No | If `true`, advertises prompt capabilities to MCP clients. |
| hasResources | bool | No | If `true`, advertises resource capabilities to MCP clients. |
| hasTools | bool | No | If `true`, advertises tool capabilities to MCP clients. |
| stateless | bool | No | If `true`, creates a stateless MCP server (no session tracking). Default `false` (stateful). |
| enabled | bool | No | If `true`, the server is enabled and ready to serve requests. Default `false`. |
| uri | string | No | Custom URI path for the MCP server, e.g. `"/mcp/my-server"`. Used for routing incoming MCP requests. |
| uriRegex | string | No | Custom regex pattern for matching request URIs to this server. Used instead of `uri` for flexible routing. |

#### MCPTool
<a name="mcptool"></a>

|Field|Data Type|Required|Description|

| uri | string | No | Custom URI subpath for this tool. Defaults to `"/<tool-name>"` (lowercased). |

#### ToolBehavior
<a name="toolbehavior"></a>

Configures what the tool does when invoked. Exactly one flag should be set to `true`.

|Field|Data Type|Required|Description|
|---|---|---|---|
| ping | bool | No | Returns a simple ping response. |
| echo | bool | No | Echoes back the tool call arguments. |
| time | bool | No | Returns the current server time. |
| stream | bool | No | Streams the configured response payload back in chunks. Uses `config.streamCount` and `config.delay`. |
| elicit | bool | No | Triggers an elicitation reverse invocation to the MCP client during tool execution. |
| sample | bool | No | Triggers a createMessage (sampling) reverse invocation to the MCP client during tool execution. |
| listRoots | bool | No | Requests the MCP client's root list during tool execution. |
| fetch | bool | No | Fetches data from a remote HTTP endpoint (non-MCP). Requires `config.remote` with a valid URL. |
| remote | bool | No | Forwards the tool call to a remote MCP server's tool. Requires `config.remote`. |
| multiRemote | bool | No | Forwards the tool call to multiple remote MCP server tools in sequence. Requires `config.multiRemote`. |
| resumable | bool | No | Makes the tool call resumable, supporting progress notifications. |
| agents | bool | No | Calls an A2A (Agent-to-Agent) agent. Requires `config.agent`. |
| serverDetails | bool | No | Returns the current MCP server's details. Auto-added to every server. |
| serverPaths | bool | No | Returns all registered server URI paths. Auto-added to every server. |
| allServers | bool | No | Returns details of all MCP servers on the port. Auto-added to every server. |
| allComponents | bool | No | Returns all registered components from all servers. Auto-added to every server. |

#### Delay
<a name="delay"></a>

|Field|Data Type|Description|
|---|---|---|
| min | duration | Minimum delay duration (e.g. `"100ms"`, `"1s"`). |
| max | duration | Maximum delay duration. A random value between `min` and `max` is used. |
| count | int | Number of times to apply the delay. |

When used as a path parameter string (e.g. in `/delay={delay}`), the format is `"<min>-<max>"` or `"<min>-<max>:<count>"`, e.g. `"100ms-500ms"` or `"1s-2s:5"`.

#### Status Path Parameter Format

The `{status}` path parameter in `/mcpapi/status/set/{status}` accepts the following formats:
- Single status code: `500`
- Multiple comma-separated codes: `404,503` (cycled through on successive requests)
- With repeat count: `500:10` (apply 10 times, then revert)
- Combined: `404,503:5` (cycle through 404 and 503, apply 5 times total)
